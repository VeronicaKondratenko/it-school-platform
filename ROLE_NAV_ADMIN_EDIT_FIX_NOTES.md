# Role navigation and admin user edit fix

Виправлено:

1. `questions.html` тепер доступний тільки студенту. Якщо цю сторінку відкриє адміністратор або викладач, система автоматично перенаправить його на відповідний розділ: `admin-questions.html` або `teacher-questions.html`.
2. Спільні сторінки більше не показують адміну/викладачу студентські пункти меню `Оцінки`, `Завдання`, `Матеріали`. Пункт `Питання/Звернення` автоматично веде на правильний модуль залежно від ролі.
3. Прибрано згадку про старий розділ повідомлень у `teacher-dashboard.html`.
4. Адмінське редагування користувача посилено на backend: endpoint `/api/admin/users/{id}` явно приймає JSON body, нормалізує поля та гарантовано записує `full_name`, email, телефон, дату народження, фото та роль.
5. Frontend після збереження користувача оновлює локальний стан і повторно завантажує список користувачів.

Після застосування патча потрібно задеплоїти і backend, і frontend.
