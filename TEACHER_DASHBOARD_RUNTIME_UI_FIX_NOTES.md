# Teacher dashboard runtime/UI fix

Виправлено проблему, коли `teacher-dashboard.html#grades` або інші вкладки відкривалися, але сторінка залишалась на огляді зі спінерами.

Причина: початкова ініціалізація чекала кілька API-запитів перед переходом на потрібну секцію. Якщо один із допоміжних запитів затримувався або падав, розділ не перемикався, а огляд залишався у стані завантаження.

Що змінено:
- потрібна секція з `#grades`, `#students`, `#courses`, `#attendance` відкривається одразу;
- допоміжні запити виконуються незалежно через `Promise.allSettled`, тому один збій більше не блокує сторінку;
- активний пункт sidebar підсвічується і для кнопок, і для посилань виду `teacher-dashboard.html#grades`;
- додано обробку `hashchange`;
- legacy badge більше не падає, якщо старий inbox-елемент відсутній;
- підсилено стилі світлої теми для сторінки викладача, щоб не було суміші світлого фону з темними картками та невидимим текстом.

Після оновлення потрібно перезапустити frontend на Render і зробити Ctrl+F5.
