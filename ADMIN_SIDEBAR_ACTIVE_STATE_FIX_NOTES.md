# Admin sidebar active state fix

Виправлено проблему, коли після переходу з одного пункту адмін-меню на інший відкривалась правильна секція, але підсвіченим залишався попередній пункт.

Причина: `showSection()` оновлював активний пункт меню до оновлення `location.hash`. Допоміжна функція `updateRoleSidebarActive()` читала старий hash, тому повертала підсвітку на попередній пункт, наприклад `#courses`.

Виправлення: у `frontend/admin-dashboard.html` hash тепер оновлюється перед викликом `showSection()`, після чого активний стан меню примусово синхронізується з поточною секцією.
