# Double-click navigation fix

Виправлено ситуацію, коли в адмін-панелі або викладацькій панелі перший клік по пункту бокового меню міг лише змінити стан/хеш або чекати стартового завантаження, а фактичний розділ відкривався тільки після повторного кліку.

## Що змінено

- Навігація адмін-панелі привʼязується до кліків одразу після `DOMContentLoaded`, до очікування профілю та довгих API-запитів.
- Перше видиме перемикання секції виконується миттєво через `showSection(...)`, а дані догружаються окремо.
- Прибрано подвійне спрацювання: inline `onclick="loadSection(...)"` залишено як fallback, але основний обробник працює в capture-фазі та блокує дублювання.
- Для викладацької панелі застосовано такий самий одноразовий обробник секцій.
- Для role-aware sidebar додано захист від повторного bind-у та обробку same-page hash navigation без дублювання подій.

## Перевірено

- `frontend/api.js` — `node --check`
- inline-скрипти `admin-dashboard.html`, `teacher-dashboard.html`, `schedule.html` та основних сторінок — `node --check`
