# Telegram bot fix notes

У цій фінальній версії виправлено проблему, коли Telegram-бот отримував update, але не відповідав і в логах було:

```text
Update id=... is not handled
```

## Що змінено

1. У `backend/app/bot/handlers.py` додано fallback-handler `fallback_message()`.
   - Бот тепер відповідає на звичайний текст, невідомі команди, фото/стікери/не текстові повідомлення.
   - Замість мовчання користувач отримує підказку з доступними командами.

2. У `backend/app/bot/handlers.py` додано fallback-handler `fallback_callback()`.
   - Якщо натиснута стара або невідома inline-кнопка, бот показує зрозуміле повідомлення.

3. Старі підказки `/link email` замінено на правильну логіку `/link КОД`.
   - Акаунт Telegram прив'язується через одноразовий код із веб-кабінету.

4. У `backend/.env` для локального запуску встановлено:

```env
ENABLE_BOT=true
BOT_MODE=polling
TELEGRAM_WEBHOOK_URL=
TELEGRAM_WEBHOOK_SECRET=
```

5. У стартових скриптах збільшено очікування порту backend `8000` до 90 секунд, щоб скрипт не показував помилку занадто рано, поки запускається бот.

## Як перевірити

1. Запустити `start (Windows).bat` або backend вручну.
2. У логах має бути:

```text
Telegram bot mode: polling
Start polling
Run polling for bot ...
```

3. Написати боту:

```text
/start
```

4. Написати звичайний текст, наприклад:

```text
привіт
```

Бот має відповісти підказкою, а в логах не повинно бути `Update is not handled` для звичайних повідомлень.

## Важливо

Файл `backend/.env` потрібен для локального запуску бота. Не завантажуйте його у відкритий GitHub-репозиторій. Для публічного репозиторію використовуйте `.env.example` і налаштовуйте секрети через змінні середовища хостингу.
