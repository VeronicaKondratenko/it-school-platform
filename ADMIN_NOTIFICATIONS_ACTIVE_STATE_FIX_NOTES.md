# Admin notifications active state fix

Виправлено помилку, коли після переходу на розділ `#notifications` у контенті відкривалась «Розсилка сповіщень», але в лівому меню залишався підсвіченим пункт «Звіти».

Що змінено:
- для кнопок адмінського sidebar додано явний `data-section`;
- активний пункт меню тепер синхронізується з реально відкритим розділом, а не лише з попереднім hash/onclick;
- після асинхронного завантаження розділу активний стан додатково перевіряється у `finally`;
- глобальний `updateRoleSidebarActive()` у `api.js` також враховує `data-section`.
